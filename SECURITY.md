# Security Policy

## Supported Versions

Currently, only the latest deployment of ScholarMe is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

ScholarMe takes security very seriously. If you discover a vulnerability in the application, infrastructure, or any connected services, please responsibly disclose it to our team.

**DO NOT post vulnerability details on public issue trackers or forums.**

Instead, please email the details to our security team:
**security@scholarme.com** (or to the primary project maintainer).

### What to Include in Your Report
To help us triage and resolve the issue quickly, please include:
1. **Description**: A summary of the vulnerability and its potential impact.
2. **Reproduction Steps**: Detailed instructions or a Proof of Concept (PoC) to reproduce the issue.
3. **Environment**: Where the issue was found (e.g., API endpoint, specific UI component, database).
4. **Suggested Fix**: If you have a proposed mitigation or fix, please include it.

### Our Response Process
- We will acknowledge receipt of your vulnerability report within 48 hours.
- We will provide an estimated timeline for remediation based on severity.
- We will notify you when the vulnerability is patched and deployed.
- With your permission, we will acknowledge your contribution in our security release notes.
